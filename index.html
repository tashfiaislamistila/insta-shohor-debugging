Question & Answer

Q1: How does javascript works?

Ans: Everything in javascript happens inside an ececution context.Execution context is like a big box which is contain two components.First components is memory component and second component is the code component.In memory component all the variable and function are stored as key-value pairs.And code component is the place where code is executed one line at a time.Javascript is a synchronous single threaded language.Javascript can only execute one command at a time.It is execute in a specific order.So that means it can only go to the next line once the current line has been finished executing.

Q2: Difference between local storage and session storage??

Ans: Session Storage: Session storage data has a larger capacity to hold.When we stored data in session storage that means the data is continue to exists only for that particular session.If the user close the tab data is lost session storage has a expiry.But session storage has larger capacity to hold data.Capacity of session storage 5 MB, and also session storage store data in client side.

Local Storage: In local storage it does not clean data itself.Local storage does not come with expiary. If the user closes the web browser tab close or shut down the system and come back again to visit same web browser or website data is still exists at the browser. People can keep data stored in the browser till as long as user want.Local storage has highest memory capasity compare to session storage.Local storage memory capacity is depend on what type of device or browser you used.
